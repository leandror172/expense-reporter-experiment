# cmd/workbook-inspect — quick notes

## Purpose
Thin CLI wrapper over **`internal/inspect`** (extraction core lifted there in Phase G1,
session 29) — dumps the full structural map of an Excel workbook to JSON. The library is also
used by `test/verify.WorkbookStructureMatches` (generate-workbook acceptance) — JSON schema is
therefore a test contract; output verified byte-identical at the G1 refactor.

## Usage
```
go build -o /tmp/workbook-inspect ./cmd/workbook-inspect/
/tmp/workbook-inspect <workbook.xlsx> <output-dir>
# → output-dir/manifest.json + one <Sheet>.json per sheet
```
Takes the workbook path as an arg — does NOT read config.json (no config dependency).

## Output schema (per sheet JSON)
`sheet`, `dimensions`, `columnWidths`, `rowHeights`, `mergedCells[]`, `crossSheetRefs[]`,
`rows[]`. Each row: `row`, `rowType`, optional `rowFill`, `cells[]`.
Each cell: `col`, `value`, `formula`, `style{bgColor,bold,border{Top,Bottom,Left,Right}}`.

## Key design facts
- **Cell inclusion:** a cell is emitted if it has a value, a formula, OR a non-default style
  (keeps styled-but-empty cells). Iterates `GetRows` output → captured only up to the last
  valued cell per row (trailing empties dropped) — NOT a complete used-range style map.
- **`rowType` classifier:** text/formula signals first (month names, Item/Valor, "Total", SUM),
  palette as corroboration — so it degrades on off-palette sheets. Types: header-month,
  header-col, total-row, data-row, category-label, separator.
- **`rowFill` / separator rows:** otherwise-empty rows carrying a ROW-LEVEL fill (black
  separator bands) are resurrected by probing an empty cell's style (GetCellStyle falls back to
  the row style). Added commit `906a99d`.
- **crossSheetRefs:** parsed from FORMULAS (not values), handling `'Quoted Name'!Ref`. ODS
  dot-notation strings (`'Listas de itens'.F37` inside CONCATENATE) are NOT counted — they're
  constructed strings, not real refs.
